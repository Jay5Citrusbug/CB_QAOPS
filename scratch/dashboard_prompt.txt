# Dashboard Redesign Prompt

Redesign the QA Portal Dashboard with a clean, modern, professional SaaS-style layout. The dashboard should focus on task visibility and daily productivity.

## Layout Structure

### Header Section

Display a personalized greeting:

**Good Morning, Jay!**
*Here's what's happening today in your QA portal.*

---

## Row 1 (Two Equal Columns)

### Left Card – Today's Tasks

Title: **Today's Tasks**

Purpose:
Display all tasks assigned to the logged-in user that are due today from the Task Board.

For each task show:

* Task Name
* Project / WorkPod Name
* Priority Badge (High, Medium, Low)
* Status Badge (Open, In Progress, Review, Completed)
* Due Time
* Quick View button

Example:

```
Today's Tasks

[High] Login API Validation
Project: CB QOps
Status: In Progress
Due: 4:00 PM

[Medium] Regression Testing
Project: Student Portal
Status: Open
Due: 6:00 PM
```

If no tasks exist:

```
No tasks assigned for today.
```

Include:

* View All Tasks link
* Task count badge in header

---

### Right Card – Today's Status

Title: **Today's Status**

Purpose:
Display the latest daily status submitted by the user.

Show:

* Today's status summary
* Submission timestamp
* Current submission status
* Edit Status button
* Submit Status button (if not submitted)

Example:

```
Today's Status

Completed smoke testing for Doc-to-Task feature.
Validated task creation workflow.
Logged 3 bugs.

Submitted at: 02:39 PM
Status: Submitted
```

Status Badge Colors:

* Submitted → Green
* Draft → Orange
* Pending → Gray

If no status submitted:

```
No status submitted today.
```

Display a prominent:
**"Add Today's Status"** button.

Add summary widgets above the cards:

* Total Tasks Today
* Open Tasks

Use compact KPI cards with icons.

The dashboard should immediately help a QA engineer understand:

1. What tasks need attention today.
2. Whether today's status has been submitted.
3. What upcoming work is planned.

Focus on productivity, visibility, and quick action accessibility.
